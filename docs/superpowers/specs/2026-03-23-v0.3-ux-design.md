# Cliphop v0.3 — Better UX Design

**Date:** 2026-03-23
**Status:** Approved

---

## Overview

v0.3 delivers three milestones (configurable hotkey, search/filter, pin entries) plus a set of baseline UX polish fixes carried forward from the v0.2 audit. The centerpiece change is replacing the native `NSMenu` popup with a custom `NSPanel`, which is required to support live search and right-click interactions.

---

## UX Fixes (baseline polish)

These apply regardless of the milestone they belong to:

| Surface | Fix |
|---|---|
| Tray | `"Settings…"` → `"Settings"` (remove ellipsis) |
| Tray | Clear History: no confirmation dialog — execute immediately |
| Tray | History items are now **clickable** (paste on click, same as selecting from popup) |
| Popup | Remove `"0: "` index prefix from item labels |
| Popup | Increase label truncation from 30 to 60 characters |
| Settings | `"Launch at login"` moved into a new **General** section (was orphaned between Accessibility and History) |
| Settings | Log file path gains an **Open** button (reveals file in Finder) |

## Milestone 1 — Configurable Hotkey

### User-facing behaviour

- Settings dialog shows a new **General** section containing:
  - `Launch at login` checkbox (moved from its orphaned position)
  - `Hotkey` row: displays current hotkey as a key-cap badge (e.g. `⌥V`) + a **Record…** button
- Clicking **Record…** enters recording state:
  - Badge text changes to `"Press shortcut…"` with a pulsing blue border
  - Button changes to **Cancel**
  - The next modifier+key combo pressed is captured and saved
- On capture: hotkey takes effect immediately (no need to close Settings); badge updates to new combo
- Default: `Option+V`
- Constraint: must include at least one modifier key (⌘ ⌥ ⌃ ⇧)

### Persistence

`~/.cliphop/config` gains a `hotkey` field:

```
hotkey=option+v
```

Serialisation format: modifier names separated by `+`, lowercase, key last (e.g. `option+v`, `ctrl+shift+v`).

### Architecture

- `config.rs`: add `hotkey: String` field (default `"option+v"`)
- `hotkey.rs`: add `re_register(new_combo: &str) -> Result<(), HotkeyError>` — unregisters the current hotkey and registers the new one
- `settings.rs`: add hotkey recorder widget using an `NSTimer`-based recording state; applies `hotkey::re_register` on capture and saves config

## Milestone 2 — Search/Filter in Popup

### User-facing behaviour

The `NSMenu` popup is **replaced** by a custom borderless `NSPanel`.

**On open (Option+V):**
- Panel appears at cursor position
- A search field is present at the top and immediately focused — the user can start typing with zero clicks
- History items are listed below the search field, most recent first
- The most-recent item has a subtle blue tint and a `"most recent"` label

**While browsing (no query):**
- History items are shown in full
- Pinned shelf is visible at the bottom (see Milestone 3)
- Left-click on an item: pastes it and closes the panel
- Right-click on an item: shows context menu (see below)
- ↑↓ arrows navigate the list; Enter selects the focused item
- Esc closes the panel without selection

**While searching (query typed):**
- List filters live to items containing the query (case-insensitive substring match)
- Matching characters are highlighted in yellow within each result
- Result count shown at the bottom of the filtered list (e.g. `— 2 results —`)
- Esc clears the query (returns to browsing); second Esc closes the panel
- Pinned shelf remains visible and unfiltered at all times

**Right-click context menu (any item):**

| Item type | Menu options |
|---|---|
| History item | Pin · Delete from history |
| Pinned item | Unpin · Delete |

**Empty state:**
- When history is empty and no pinned items exist, panel shows a centred message: `"No clipboard history yet"`
- When a search query returns no results, shows: `"No matches for "query""`

### Architecture

- `popup.rs`: full rewrite
  - `NSPanel` (borderless, floating, `NSWindowStyleMaskBorderless`, `NSFloatingWindowLevel`)
  - `NSTextField` search field as subview, first responder on open
  - Custom item row views with hover highlight, left-click and right-click handling
  - Pinned shelf as a fixed bottom section with purple tint and `PINNED` header label
  - Panel dismisses on: Esc, click outside (loses key window status), or item selection
- `ClipboardHistory` in `clipboard.rs` gets `pin(idx)`, `unpin(idx)` methods (see Milestone 3)

## Milestone 3 — Pin Entries

### User-facing behaviour

- Pinned entries are snippets that **never expire** and are not subject to the `max_history` cap
- Pinning: right-click any history item in the popup → **Pin**
- Unpinning: right-click any pinned item → **Unpin**; the item moves back into regular history at position 0
- Pinned items appear in the **pinned shelf** at the bottom of the popup panel (always visible, unaffected by search)
- Pinned items also appear in the **tray menu** in a dedicated `PINNED` section above the separator
- Pinned items are clickable in the tray (same paste behaviour as history items)
- There is no explicit limit on the number of pinned items

### Data model

`HistoryEntry` in `history.rs` gains a `PinnedText` variant:

```rust
pub enum HistoryEntry {
    Text(String),
    PinnedText(String),
}
```

`ClipboardHistory` in `clipboard.rs` tracks pinned items in a separate `VecDeque<String>`:
- `pin(idx: usize)` — moves item from history to pinned list
- `unpin(idx: usize)` — moves item from pinned list to front of history
- `pinned_items() -> &VecDeque<String>` — read access for rendering
- Deduplication: re-copying a pinned item does not add it to history

### Persistence

`history::save_all` is updated to serialise both `Text` and `PinnedText` variants. `history::load` deserialises both. Format is unchanged except for a new entry type marker.

### Architecture

- `history.rs`: handle `HistoryEntry::PinnedText` in `save_all` / `load`
- `clipboard.rs`: add `pin`, `unpin`, `pinned_items` to `ClipboardHistory`; adjust `poll` dedup to skip pinned items
- `popup.rs`: wire right-click context menu → `pin` / `unpin` calls; render pinned shelf
- `tray.rs`: render `PINNED` section from `history.pinned_items()`
- `main.rs`: pass pinned items to `update_tray`

## Module Change Summary

| Module | Change |
|---|---|
| `popup.rs` | Full rewrite: `NSPanel` with search field, item rows, right-click menus, pinned shelf |
| `hotkey.rs` | Add `re_register(combo)` for runtime hotkey swapping |
| `history.rs` | Add `HistoryEntry::PinnedText`; update `save_all` / `load` |
| `clipboard.rs` | Separate pinned `VecDeque`; add `pin`, `unpin`, `pinned_items`; adjust dedup |
| `tray.rs` | Clickable items; `PINNED` section; label format cleanup; `"Settings"` label fix |
| `settings.rs` | New `General` section with hotkey recorder; `Open` button for log path |
| `config.rs` | Add `hotkey: String` field |
| `main.rs` | Pass pinned items to tray; wire hotkey re-registration from settings |

## Out of Scope for v0.3

- Image or file clipboard entries (v0.4)
- Paste transformations (v0.4)
- iCloud sync (v0.5)
- Folder/group support (v0.5)
