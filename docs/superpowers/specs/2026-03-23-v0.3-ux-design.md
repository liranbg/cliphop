# Cliphop v0.3 — Better UX Design

**Date:** 2026-03-23
**Status:** Approved

---

## Overview

v0.3 delivers three milestones (configurable hotkey, search/filter, pin entries) plus a set of baseline UX polish fixes carried forward from the v0.2 audit. The centerpiece change is replacing the native `NSMenu` popup with a custom `NSPanel`, which is required to support live search and right-click interactions.

---

## UX Fixes (baseline polish)

These apply regardless of the milestone they belong to:

| Surface | Fix |
|---|---|
| Tray | `"Settings…"` → `"Settings"` (remove ellipsis) |
| Tray | Clear History: no confirmation dialog — execute immediately. `confirm_and_clear_history()` in `settings.rs` is deleted; callers invoke the clear callback directly. |
| Tray | History items are now **clickable** (paste on click, same as selecting from popup) |
| Popup | Remove `"0: "` index prefix from item labels |
| Popup | Increase label truncation from 30 to 60 characters |
| Settings | `"Launch at login"` moved into a new **General** section (was orphaned between Accessibility and History) |
| Settings | Log file path gains an **Open** button (reveals file in Finder via `NSWorkspace.activateFileViewerSelectingURLs`) |

## Milestone 1 — Configurable Hotkey

### User-facing behaviour

- Settings dialog shows a new **General** section containing:
  - `Launch at login` checkbox (moved from its orphaned position)
  - `Hotkey` row: displays current hotkey as a key-cap badge (e.g. `⌥V`) + a **Record…** button
- Clicking **Record…** enters recording state:
  - Badge text changes to `"Press shortcut…"` with a pulsing blue border
  - Button changes to **Cancel**
  - The next modifier+key combo pressed is captured and saved
  - If no combo is pressed within 10 seconds, recording auto-cancels (reverts to previous hotkey)
- On capture: hotkey takes effect immediately (no need to close Settings); badge updates to new combo
- Default: `Option+V` (`alt+v`)
- Constraint: must include at least one modifier key (⌘ ⌥ ⌃ ⇧); bare key presses are ignored during recording

### Hotkey config serialisation format

Stored in `~/.cliphop/config` as:

```
hotkey=alt+v
```

**Token rules** (map directly to `global-hotkey` crate types):

| Config token | `Modifiers` flag | Display glyph |
|---|---|---|
| `meta` | `Modifiers::META` | ⌘ |
| `ctrl` | `Modifiers::CONTROL` | ⌃ |
| `alt` | `Modifiers::ALT` | ⌥ |
| `shift` | `Modifiers::SHIFT` | ⇧ |

Key tokens follow the W3C `KeyboardEvent.code` spec with the `Key` and `Digit` prefixes stripped and lowercased:
- Letter keys: `a`–`z` (maps to `Code::KeyA`–`Code::KeyZ`)
- Digit keys: `0`–`9` (maps to `Code::Digit0`–`Code::Digit9`)
- Function keys: `f1`–`f12` (maps to `Code::F1`–`Code::F12`)
- Special keys allowed: `space` (`Code::Space`), `tab` (`Code::Tab`), `comma` (`Code::Comma`)

**Canonical ordering:** modifiers in `meta+ctrl+alt+shift+<key>` order. When parsing, order is irrelevant; when serialising, always write in canonical order.

**Examples:** `alt+v`, `meta+shift+p`, `ctrl+alt+f1`

### Architecture

- `config.rs`: add `hotkey: String` field (default `"alt+v"`)
- `hotkey.rs`: add `re_register(new_combo: &str) -> Result<(), String>` — unregisters the current hotkey, parses the combo string into `(Option<Modifiers>, Code)`, registers the new `HotKey`, updates internal state
- `settings.rs`:
  - Hotkey recorder widget: an `NSTextField` (read-only, displays current hotkey) + **Record…** `NSButton`
  - On **Record…** click: install an `NSEvent` local monitor (`addLocalMonitorForEventsMatchingMask: NSEventMaskKeyDown`) to capture the next key event; start a 10-second `NSTimer` for auto-cancel. `NSEvent` local monitors are application-level and fire in all run loop modes, including `NSModalPanelRunLoopMode` used by `NSAlert.runModal` — no special mode registration is required. Pure modifier-only key events (no non-modifier `Code`) are ignored and do not reset the timer.
  - On key capture or cancel: remove the local monitor via `removeMonitor:`, invalidate the timer, update the badge, call `hotkey::re_register`, save config

## Milestone 2 — Search/Filter in Popup

### User-facing behaviour

The `NSMenu` popup is **replaced** by a custom borderless `NSPanel`.

**On open (Option+V):**
- Panel appears at cursor position
- A search field is present at the top and immediately focused — the user can start typing with zero clicks
- History items are listed below the search field, most recent first
- The most-recent item has a subtle blue tint and a `"most recent"` label (suppressed when history is empty)

**While browsing (no query):**
- History items are shown in full (60-char truncation)
- Pinned shelf is visible at the bottom (see Milestone 3)
- Left-click on an item: pastes it and closes the panel
- Right-click on an item: shows context menu (see below)
- ↑↓ arrows navigate both the history list and the pinned shelf (navigation crosses the boundary)
- Enter selects the focused item; Esc closes the panel without selection

**While searching (query typed):**
- List filters live to items containing the query (case-insensitive substring match)
- Matching characters are highlighted in yellow within each result
- Result count shown at the bottom of the filtered list (e.g. `— 2 results —`)
- Esc clears the query (returns to full browsing state); second Esc closes the panel
- Pinned shelf remains visible and unfiltered at all times
- Arrow navigation crosses into the pinned shelf even while a query is active

**Right-click context menu (any item):**

| Item type | Menu options |
|---|---|
| History item | Pin · Delete from history |
| Pinned item | Unpin · Delete |

**Empty state:**
- When history is empty and no pinned items exist: `"No clipboard history yet"` centred in panel
- When a search query returns no history results (pinned shelf still shown): `"No matches for "query""` in the history area

### Event loop

`show_popup` must spin a nested `NSRunLoop` (via `NSRunLoop::currentRunLoop().runUntilDate(...)` in a loop) while the panel is visible, exactly as `NSMenu` did internally. This keeps the main tao event loop responsive. The panel sets a flag on dismiss; the loop checks the flag and exits. `simulate_paste()` behaviour is unchanged — focus is restored to the previous app before the nested run loop unwinds.

### Architecture

- `popup.rs`: full rewrite
  - `NSPanel` (`NSWindowStyleMaskBorderless`, `NSFloatingWindowLevel`, `NSWindowCollectionBehaviorTransient`)
  - `NSTextField` search field as subview, made first responder via `window.makeFirstResponder`
  - Custom `NSView` subclass for each item row: hover highlight via `mouseEntered`/`mouseExited`, left-click via `mouseDown`, right-click via `rightMouseDown` → `NSMenu` context menu
  - Pinned shelf as a fixed bottom `NSView` with purple background tint and `PINNED` label
  - Dismiss triggers: Esc key, `windowDidResignKey` delegate callback, item selection
  - Nested `NSRunLoop` spun until `dismissed` flag is set
- `ClipboardHistory` in `clipboard.rs` gets `pin(idx)`, `unpin(idx)` methods (see Milestone 3)

## Milestone 3 — Pin Entries

### User-facing behaviour

- Pinned entries are snippets that **never expire** and are not subject to the `max_history` cap
- Pinning: right-click any history item in the popup → **Pin**
  - If the item's text is already in the pinned list, `pin()` is a no-op (silent dedup)
- Unpinning: right-click any pinned item → **Unpin**; the item moves to the front of regular history (position 0)
  - If history is already at `max_history` capacity when unpinning, the oldest history item is evicted
- Pinned items appear in the **pinned shelf** at the bottom of the popup panel (always visible, unaffected by search)
- Pinned items also appear in the **tray menu** in a dedicated `PINNED` section above the separator, and are clickable (paste on click)
- There is no cap on the number of pinned items (intentional — power-user feature)

### Data model

`HistoryEntry` in `history.rs` is the **serialisation type only** (used in `load` / `save_all` signatures). It gains a `PinnedText` variant:

```rust
pub enum HistoryEntry {
    Text(String),
    PinnedText(String),
}
```

The in-memory representation in `ClipboardHistory` (`clipboard.rs`) uses two separate `VecDeque<String>` fields:
- `items: VecDeque<String>` — regular history (unchanged)
- `pinned: VecDeque<String>` — pinned items

New methods on `ClipboardHistory`:
- `pin(idx: usize)` — moves item at `items[idx]` to the back of `pinned`; no-op if text already in `pinned`
- `unpin(idx: usize)` — moves item at `pinned[idx]` to the front of `items`; evicts `items.pop_back()` if at cap
- `pinned_items() -> &VecDeque<String>` — read access for rendering
- `poll()` dedup: if a newly copied text matches any pinned item, the copy is ignored (not added to history)

### Persistence

Pinned items are saved to the **same** `~/.cliphop/history` file as regular history, using `kind: "pinned_text"` (same encrypted format, same `n`/`d`/`v` fields as `kind: "text"`).

`save_all` signature changes to:

```rust
pub fn save_all(history: &[String], pinned: &[String])
```

All history entries are written first, then all pinned entries. On `load`, entries with `kind: "text"` become `HistoryEntry::Text`; entries with `kind: "pinned_text"` become `HistoryEntry::PinnedText`. Unknown kinds continue to be skipped.

`main.rs` splits the loaded `Vec<HistoryEntry>` into two `Vec<String>` slices before calling `ClipboardHistory::load_items` (history) and a new `ClipboardHistory::load_pinned(items: Vec<String>)`.

### Architecture

- `history.rs`: add `PinnedText` variant; update `save_all_to` to `save_all_to(path: &str, key: [u8; 32], history: &[String], pinned: &[String])` — writes all history entries first (kind `"text"`), then all pinned entries (kind `"pinned_text"`); update `load_from` to match `"pinned_text"` and return `HistoryEntry::PinnedText`
- `clipboard.rs`: add `pinned: VecDeque<String>`; add `pin`, `unpin`, `pinned_items`, `load_pinned`; adjust `poll` dedup
- `popup.rs`: wire right-click → `pin` / `unpin`; render pinned shelf from `pinned_items()`
- `tray.rs`: render `PINNED` section from `pinned_items()`
- `main.rs`: split loaded entries; call `load_pinned`; pass pinned items to `update_tray`; call `history::save_all(history, pinned)` on change

## Module Change Summary

| Module | Change |
|---|---|
| `popup.rs` | Full rewrite: `NSPanel` with nested run loop, search field, item rows, right-click menus, pinned shelf |
| `hotkey.rs` | Add `re_register(combo: &str)` for runtime hotkey swapping |
| `history.rs` | Add `HistoryEntry::PinnedText`; update `save_all(history, pinned)` and `load` |
| `clipboard.rs` | Add `pinned: VecDeque<String>`; add `pin`, `unpin`, `pinned_items`, `load_pinned`; adjust dedup |
| `tray.rs` | Clickable items; `PINNED` section; label format cleanup; `"Settings"` label fix; direct clear (no confirmation) |
| `settings.rs` | New `General` section with hotkey recorder (NSEvent local monitor + NSTimer); `Open` button for log path; remove `confirm_and_clear_history` |
| `config.rs` | Add `hotkey: String` field (default `"alt+v"`) |
| `main.rs` | Split loaded entries into history/pinned; pass pinned to tray; wire hotkey re-registration; call `save_all(history, pinned)` |
